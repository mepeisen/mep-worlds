Requirements

Before starting this tutorial you should have a look on the following requirements:

  • You need a local Eclipse installation with installed Lua Development Tools (LDT).

Although this tutorial is written for Eclipse you may adapt some of the information to other IDEs. Maybe in future there will be tutorials for some other IDEs.

What you will learn

Finishing this tutorial you will learn how to setup a project within eclipse and how to run either Minecraft or ComputerCraft simulator CCLite. You will find a way how to run Your code within Minecraft or within CCLite.

Step 1 Checkout XWOS

Within Eclipse open GIT perspective.

Paste the GitHub url of XWOS (Ctrl+V): 

http://git.xworlds.eu/mepeisen/mep-worlds.git

If you plan to contribute to XWOS itself you should first create a fork at github and then use your forks url for checkout. However you may add your forks url later on.

The "Clone Git Repository" dialog opens. Enter your Github username/password if you plan to contribute. Click "next" two times, check the settings and click "finish".

Right click on the git repository workspace and click "Import projects".

Within the import dialog you should see the XWOS source project. Click "finish".

You can now switch to the LDT perspective to start developing in LUA.

Step 3 CCLite setup

There are multiple versions of CCLite around. XWOS itself contains some lines of code to support classic ComputerCraft within Minecraft as well as CCLite. It may not be supported on a specific installation. If you find versions that do not work please create a GitHub issue. But be aware that I may decide not supporting your version.

Download and install one of the following versions:

  • For 1.7.5: Fork of Gamax92 or Fork from PixelToast (the original from Sorroko seems to be upt of date)
  • For 1.8: Fork of Wilma456

Create a new external tool within Eclipse. Select the LOVE2D executable and the CCLite LOVE file.

Launch the new external tool and ensure that it starts and can be used.

Step 4 Project builder

There are several ways to create an automatic build setup. In future versions of XWOS I will add some small eclipse plugin to setup your Minecraft or CCLite in an easy way. For now I recommend to create a small new project, let us call it build-myapp.

Now create new folders linked with CCLite and/or your minecraft resourcepack folders. Right click on your project and click on "new folder".

Select radio button "Link to alternate location" and click "browse" to select the target folder.

Now we can create a small ant build.xml. This build script will be responsible to copy the project files to your minecraft or cclite installation.

The given example will copy all XWOS files to your local CCLite installation.
