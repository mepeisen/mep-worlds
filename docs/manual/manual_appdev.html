Overview

Before we start let me explain some guidelines. First of all you should already be familiar with ComputerCraft, LUA and XWOS. If you do not have a working setup on a ComputerCraft device you should play around with XWOS as a normal user. The following chapters are not giving you any user experience. Instead a good developer always should experience the software product he/she wants to develop for.

I am a friend of clean object oriented programming. Every example is based on clean documentation and clean object writings. Why should you follow this guideline? Typically every software product grows. You may come back to your product after some months and nearly nobody ever is able to remember every single code fragment after months and years. So every clean and well-documented code helps you understanding what is going on.

API and detailed information

Getting more into details of XWOS:

  • CMR: The class system and the public class manager.
  • Moses: Third party library moses for tables, arrays and functional programming.
  • Allen: Third party library allen for string manipulation.
  • Utf8: Unicode support and how to use it.
  • LuaUnit: test driven development by using luaunit.
  • GUI: A gui library working similar to HTML documents/ JavaFX.
  • Process: Understanding multi tasking.
  • XWOS: Accessing core functionality of operating system.
