Overview

XWOS introduces some built-in tools available through class manager. I am dividing tool classes related to kernel/ kernel modules and tool classes available as optional API (libraries). Kernel related tool classes are always available without further action. But a certain API may not be available to a specific XWOS installation.

To use a kernel related class simply invoke _CMR.new. To use a specific API you may simply invoke _CMR.new too. But that may throw errors if a your application is not allowed to work with this API or if it is not even installed. See the following chapter for a clean way on requesting a specific API.

Core class: xwos.xwlist (doubly linked list)

A doubly linked list is a special list where each list member (node) holds reference to previous and next node. It has some performance benefits compared to arrays when checking for duplicate entries or getting the next/previous node.

You can use any XWOS object instance created by CMR with xwlist. There is no limitation. However currently the list only supports each object exactly once. As soon as the object is contained in a list it must not be added to other lists. This behavior will change in future releases by supporting list tags. For example, you can create multiple lists for tag "MyUniqueList" and the object can only be added once to any of this lists but it may be added to some list with other tags.
